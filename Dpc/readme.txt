# Библиотека DPC

**DPC** — основная разделяемая библиотека проекта **DTools**, содержащая ключевые классы, используемые в остальных компонентах проекта.

Логически библиотеку можно разделить на следующие модули:

* **GUI** — классы графических элементов: делегаты, виджеты, диалоговые окна.
* **Journal** — классы журнала и записей для реализации механизма логирования сообщений.
* **Sybus** — классы для взаимодействия с устройствами **DepRTU** через систему параметров протокола Sybus (установка соединения, чтение/запись параметров и т.д.).
* **Ресурсы** — общие ресурсы проекта.
* **Утилиты** — вспомогательные классы.

---

## GUI

Файлы модуля находятся в папке:

```
/dpc/gui/
```

### Подмодули

1. **/dpc/gui/delegates** — реализации классов различных делегатов.
2. **/dpc/gui/dialogs** — классы диалоговых окон, включая окна выбора типа соединения (канала) для работы с устройством по протоколу Sybus.
3. **/dpc/gui/widgets** — классы виджетов на основе стандартных виджетов **Qt**.
   Реализованы как простые одноэлементные виджеты, так и сложные составные (например, виджет журнала или табличный виджет со встроенной фильтрацией и сортировкой).

**Зависимости:**

* Диалоговое окно выбора канала взаимодействия с устройством зависит от модулей **Journal** и **Sybus**.
* Виджет отображения журнала зависит от модуля **Journal**.

---

## Journal

Файлы модуля находятся в папке:

```
/dpc/journal/
```

**Описание:**
Класс журнала реализован как **синглтон** и предоставляет интерфейс для логирования сообщений различных типов:

* информация
* отладка
* предупреждение
* ошибка

---

## Sybus

Файлы модуля находятся в папке:

```
/dpc/sybus/
```

Содержит все классы, связанные с взаимодействием с устройством по протоколу **Sybus**, системой параметров устройств, типами данных этих параметров и конкретными реализациями каналов связи.

### Основные классы

* **Channel** — последовательный режим работы, блокирующие вызовы: выполнение программы приостанавливается до получения ответа или достижения таймаута.
* **AsyncChannel / AsyncChannelManager** — асинхронный режим работы: очередь запросов к устройству, неблокирующий режим.

## Ресурсы

Ресурсы **не находятся** в папке проекта и берутся из каталога на уровень выше:

```
../dconf_res/
```

Содержит все иконки, используемые в DTools. Все иконки компилируются в библиотеку и доступны через **DPC** в любом приложении DTools.

---

## Утилиты

Файлы модуля находятся в папке:

```
/dpc/
```

Содержат различные вспомогательные классы, используемые в других частях библиотеки.
